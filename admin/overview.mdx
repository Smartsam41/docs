The **Admin** section is where warehouse administrators configure CoreX's behavior, rules, and integrations. Most settings here need to be configured once and then rarely touched — but they drive everything that happens in the warehouse day to day.

<Warning>
  Admin access is restricted to users with the **Admin** role. Most settings in this section affect live operations — changes take effect immediately. Test configuration changes in a non-production environment when possible, or make changes during off-peak hours.
</Warning>

## Recommended setup order

If you're setting up CoreX from scratch, work through Admin configuration in this order to avoid dependency issues:

<Steps>
  <Step title="Company Profile">
    Set your name, address, and time zone first — other settings depend on it.
  </Step>
  <Step title="Locations and Zones">
    Define your warehouse location structure (Inventory → Locations).
  </Step>
  <Step title="System Parameters">
    Set global defaults: units of measure, tolerance thresholds, and operational flags.
  </Step>
  <Step title="Sequences">
    Configure auto-numbering formats for receiving orders, shipping orders, and other documents.
  </Step>
  <Step title="Carriers">
    Connect your carrier accounts (Admin → Carriers).
  </Step>
  <Step title="Putaway and Allocation Rules">
    Configure where product goes after receiving and how inventory is reserved for orders.
  </Step>
  <Step title="Cartonization Config">
    Add your carton library if you're using automated carton selection.
  </Step>
  <Step title="Team Members">
    Add your users and assign roles (Admin → Team Members).
  </Step>
  <Step title="Accounts and Rate Cards">
    Set up shipper accounts (Customers → Accounts) and their rate cards (Billing → Rate Cards).
  </Step>
  <Step title="Integrations">
    Connect channels (Shopify, Amazon) and EDI partners (Admin → Integration Store, EDI Processing).
  </Step>
</Steps>

## Audit logging

CoreX maintains a complete audit log of all Admin configuration changes. Navigate to **Admin → Data Tools → Audit Log** to review:

- What setting was changed
- What the old value was and what the new value is
- Which user made the change
- When the change was made

The audit log is retained for 2 years and is read-only — it cannot be edited or deleted.
