The Shipping module covers the full outbound fulfillment workflow: from the moment an order enters CoreX (via channel integration, EDI, API, or manual entry) to the moment a carrier scans it on a truck. Every step is tracked with timestamps, user IDs, and scan confirmations.

## The outbound flow

<Steps>
  <Step title="Order import">
    Shipping orders enter CoreX from connected channels (Shopify, Amazon, EDI, API) or are created manually. Each order belongs to a shipper account and contains ship-to address, line items, and service requirements.
  </Step>
  <Step title="Inventory allocation">
    CoreX reserves inventory against each order line using **Allocation Rules** (FIFO, FEFO, lot-specific, or custom). Allocated orders move to **Ready** status. Orders that can't be fully allocated enter **Backordered** status.
  </Step>
  <Step title="Wave planning">
    Ready orders are grouped into **waves** — batches released to the floor together. Waves can be built manually by a supervisor or auto-generated by CoreX based on carrier cutoff times. A wave generates a pick list and creates tasks in the Task Queue.
  </Step>
  <Step title="Pick">
    Pickers work through the **Task Queue** using the **Scan to Pick** mobile flow. CoreX directs each picker to the exact location and quantity to pick, in an optimized path through the warehouse.
  </Step>
  <Step title="Pack">
    Packed product moves to a pack station. Packers use **Scan to Pack** to scan each item into a carton. CoreX verifies the right items are in the right box and can automatically select the optimal carton size via **Cartonization**.
  </Step>
  <Step title="Rate and label">
    After packing, CoreX rates the shipment against connected carrier accounts, applies any carrier rules or account preferences, and generates the carrier label. The label is printed directly to the assigned pack station printer.
  </Step>
  <Step title="Ship out">
    Labeled cartons are staged to the outbound dock. A supervisor closes out the **manifest** for each carrier, generating a manifest report and end-of-day scan. Carriers pick up and scan. The order moves to **Shipped**.
  </Step>
</Steps>

## Shipping order statuses

| Status | Description |
|--------|-------------|
| **Pending** | Order received, awaiting inventory allocation |
| **Backordered** | Insufficient inventory to fully allocate |
| **Ready** | Fully allocated and eligible for waving |
| **Waved** | Included in a released wave; pick tasks generated |
| **Picking** | At least one pick task is in progress |
| **Picked** | All items picked, awaiting packing |
| **Packing** | Packing in progress |
| **Packed** | All items packed; ready for labeling |
| **Labeled** | Carrier label printed |
| **Staged** | Staged to outbound dock, awaiting carrier pickup |
| **Manifested** | Included in a closed carrier manifest |
| **Shipped** | Carrier has confirmed pickup |
| **Cancelled** | Order cancelled before shipment |
| **Exception** | Order requires supervisor review |

## Smart Lanes

**Smart Lanes** is CoreX's intelligent routing engine for outbound orders. It evaluates each order against your configured carrier rules, rate cards, and service commitments to automatically select the best carrier and service level — balancing cost, transit time, and delivery reliability.

Smart Lanes can be configured to:
- Prefer cheapest carrier within a transit time window
- Enforce account-specific carrier preferences
- Route oversized or heavy shipments to freight automatically
- Apply DIM weight rules and carrier surcharge logic

## Ship Out Dashboard

The **Ship Out Dashboard** (Ship → Ship Out Dashboard) gives supervisors a real-time view of outbound operations:

- Orders in each status (waved, picking, packing, staged, manifested)
- Carrier cutoff countdowns — red when within 30 minutes
- Pick completion rate by wave
- Pack station productivity
- Label failures and exceptions requiring attention

## Carrier manifests

At the end of each carrier pickup window, a supervisor closes the manifest for that carrier:

1. Go to **Ship → Manifests** and find the open manifest for the carrier.
2. Review the list of packages included.
3. Click **Close Manifest**. CoreX generates a manifest report (PDF or EDI 214) and marks all packages as manifested.
4. Print the manifest report and hand it to the carrier driver.

Packages not included in a closed manifest remain in **Staged** status for the next manifest cycle.

<Warning>
  Always close manifests before the carrier's end-of-day scan. Packages not manifested may not be tracked by the carrier and could cause delivery failures.
</Warning>
